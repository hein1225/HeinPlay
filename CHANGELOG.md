## 1.1.4

### 变更

- **Windows 版本播放器统一**
  - Windows 端只保留 `flutter_mpv` 一个播放器后端，不再提供其他后端切换，避免多后端兼容性问题。
  - 清理了与外部播放器相关的实验性代码和依赖。

### 构建产物与适用设备

| 文件名 | 适用设备 | 系统要求 | 说明 |
| --- | --- | --- | --- |
| `heinplay-1.1.4-tv.apk` | Android TV / 电视盒子 | Android 7.0+（API 24+） | 横屏 Leanback 设计，遥控器焦点导航。 |
| `heinplay-1.1.4-mobile.apk` | Android 手机 / 平板 | Android 7.0+（API 24+） | 竖屏触屏 UI，支持手势与屏幕旋转。 |
| `heinplay-1.1.4-windows-portable.zip` | Windows 10/11 电脑 | Windows 10 1809+ | 解压即用，使用 `flutter_mpv` 播放。 |

### 播放器后端现状

- **Android TV / Android 手机 / 平板**：默认使用 **ExoPlayer**，可在设置中切换为 **flutter_mpv**。
- **Windows**：仅使用 **flutter_mpv**，不支持切换其他后端。

> 当前 Android 构建配置：`minSdk 24`（Android 7.0），`targetSdk 36`。

## 1.1.1

### 新增 / 优化

- **LunaTV 搜索体验增强**
  - 搜索超时由 8 秒延长至 30 秒，匹配 Selene 项目配置，解决长篇动漫（如《名侦探柯南》）搜索结果集体积过大导致的超时失败。
  - `SearchService.generateSearchVariants` 新增 `fuzzy` 模式：模糊搜索时自动识别并剥离「年番/季/部/集/篇/章」等后缀，生成基础标题变体，提升带后缀标题（如「凡人修仙传 年番4」）的命中率。
  - 详情页搜索源改为增量显示：通过 `onProgress` 回调每完成一个搜索变体即刷新界面，搜到一个源立刻展示，显著减少搜索等待与黑屏时间。
  - 新增过滤诊断日志：当 LunaTV 返回结果但相关性过滤后为空时，打印 TOP5 得分结果，便于排查过滤阈值问题。

- **详情页信息扩展**
  - 影片介绍区域新增导演、编剧、主演、首播/上映日期。
  - 新增地区、语言、集数、时长等标签展示，结构更接近 LunaTV 详情页。

- **分类页海报墙重构**
  - 海报卡片宽高比调整，一屏可完整显示两行海报。
  - 豆瓣评分显示在海报右上角，标注「豆瓣」；动漫「每日放送」使用 Bangumi 评分并显示在右上角，标注「Bangumi」。

- **品牌视觉**
  - 顶部导航栏「海因影视」标题改为 Logo 风格：加大字号、加重字重、增加字间距与红色发光阴影。

### 变更

- **详情页布局紧凑化**
  - 海报尺寸从 `280×420` 减小至 `210×320`。
  - 影片简介展示行数从 4 行调整为 2 行，为新增信息腾出空间。
  - 选集区域与播放源区域间距收紧，播放源选择区高度降低。

- **评分展示规则**
  - 动漫「每日放送」条目详情页优先展示豆瓣信息；豆瓣信息缺失时回退到 Bangumi。
  - 详情页评分区同时展示豆瓣评分与 Bangumi 评分（后者以粉色徽章区分）。

### 修复

- **播放记录刷新**
  - 详情页退出播放后，播放记录立即刷新，无需退回首页。
  - 修复 `_loadPlayRecord` 在 `await` 期间 widget 被 dispose 后调用 `setState` 导致的 `Null check operator used on a null value` 异常。

- **分类标签焦点**
  - 焦点在分类标签上时，按下键仅移动到当前分类的第二行（筛选/子分类/星期），不再误切换分类。
  - 分类切换统一由 `FocusableWidget.onTap` 处理（确认键/鼠标点击）。

- **去广告开关状态**
  - 修复设置页 M3U8 去广告开关初始状态与实际存储值不一致的问题。
  - `AdFilterEngine` 增加真实开关值日志，便于排查状态。

## 1.1.0

### 新增

- **动漫分类焦点重构**
  - 「番剧」「剧场版」按下键时焦点正确移动到筛选行第一个按钮（标签/地区/年代/排序等），不再误切换分类。
  - 新增自定义 `_DimensionButton` 控件，直接使用 `Focus` 管理焦点，绕过 `FocusableWidget`/`FocusableActionDetector` 在动态重建时 FocusNode 无法挂载的问题。
  - 「每日放送」星期行保持原有逻辑不变，支持完整左右循环。

### 变更

- **电影分类排序精简**
  - 电影分类「全部」移除「综合排序」选项，仅保留「近期热度」「首映时间」「高分优先」。

- **默认排序调整**
  - 电影、电视剧、综艺「全部」以及动漫「番剧」「剧场版」默认排序改为「近期热度」（`U`）。
  - 同步调整 `DoubanService` 排序参数处理逻辑，确保「近期热度」正确透传。

### 修复

- **海报墙上键焦点层级**
  - 修复分类页（电影、电视剧、综艺、动漫）与首页在海报墙/内容区按住上键不放时，焦点直接跳到「我的」的问题。
  - `TvShell` 根 Focus 的上键兜底逻辑现在同时处理 `KeyDownEvent` 与 `KeyRepeatEvent`，确保长按遥控器时每一帧都回到当前页面对应的顶部导航项。

- **启动时权限请求崩溃**
  - 修复 `main()` 中未初始化 Flutter Binding 就调用 `StorageService.requestStoragePermission()` 导致的 `Null check operator used on a null value` 异常。
  - 在 `main()` 开头调用 `WidgetsFlutterBinding.ensureInitialized()`，并为权限请求添加 `.catchError()` 兜底。

## 1.0.3

### 新增

- **M3U8 本地去广告**
  - 参考 TVBox 广告过滤思路，在客户端本地完成 M3U8 广告过滤，不再依赖 LunaTV 服务端广告配置。
  - 新增 `M3u8AdFilter`：支持少数 URL 过滤、不连续片段组分析、SCTE-35/CUE 广告标记识别、URL/域名特征匹配及安全回退机制。
  - 新增 `LocalM3u8Proxy`：本地 HTTP 代理服务，统一补全请求头（Referer、Origin、Range 等），兼容 ExoPlayer / media_kit / video_player 三种播放器后端。
  - 播放器后端自动识别本地代理地址，避免对 `127.0.0.1` 进行二次 LunaTV 代理包装。
  - 设置中保留去广告开关，默认关闭，开启后对所有 M3U8 播放器生效。

- **分类页默认排序调整**
  - 电影、电视剧、综艺、动漫分类中，除「近期热门」外，所有小分类（含「全部」）默认排序改为「首映时间」。
  - 切换小分类时自动重置为默认排序；「近期热门」仍保持排序维度隐藏。

### 修复

- **播放失败自动切换源**
  - 设置页将「自动切换源」开关与等待时间选项合并为同一卡片，关闭开关时隐藏等待时间选项。
  - 修复等待时间设置未真正生效的问题：现在以设置时长为总上限，等待播放器报告有效时长后才视为成功，超时则触发切换源。

- **继续播放与播放记录**
  - 海报下方不再显示年份，改为显示该条记录对应的播放源名称。

- **二维码登录**
  - 修复退出登录后再次使用二维码登录时提示「发送失败」的问题；`RemoteInputService` 单例生命周期内保持 StreamController 可用。

- **更新弹窗**
  - 弹窗尺寸加大，完整显示更新标题与日志内容。
  - 下载进度条固定在底部始终可见，不再随日志滚动到下方。

- **关于页**
  - 开源仓库地址改为国内 GitCode 地址 `https://gitcode.com/gcw_QbmhmbO8/HeinPlay`。

### 变更

- 应用版本号统一更新至 `1.0.3`（`pubspec.yaml`、关于页、`UpdateService`、`LunaTV User-Agent`）。
